# 《JavaWeb开发技术》期末综合大作业报告

**题目：图书馆借还系统**

---

## 一、项目背景

随着信息技术的飞速发展和互联网的普及，传统的图书馆管理模式已难以满足现代读者对便捷、高效服务的需求。传统的人工借阅方式存在管理效率低下、信息查询不便、数据统计困难等突出问题。开发一套基于Web的图书馆借还管理系统，将图书管理、借阅归还、读者管理等工作实现信息化和网络化，对于提升图书馆服务质量和工作效率具有重要意义。

本项目旨在设计并实现一个基于JavaWeb技术的图书馆借还系统。系统采用B/S（Browser/Server）架构，前端使用JSP技术构建动态网页，后端基于Servlet进行业务逻辑处理，使用JDBC技术连接MySQL数据库进行数据持久化存储。系统实现了用户登录注册、图书浏览搜索、图书借阅归还、后台管理等核心功能，支持读者和管理员两种角色的权限区分。

在项目开发前，本人查阅了相关的技术文献和资料。主要参考了《Java Web程序设计教程》（清华大学出版社，李兴华编著）、《MySQL数据库应用与实践》（人民邮电出版社）以及Oracle官方发布的Java Servlet 4.0规范文档。通过对这些资料的学习，对Servlet生命周期、JSP内置对象、MVC设计模式、JDBC数据库编程等核心技术有了深入理解，为项目的顺利开发奠定了理论基础。

## 二、需求分析

### 2.1 功能需求

本系统包含两类用户角色：**读者**和**管理员**，各自具有不同的功能权限。

#### 2.1.1 读者功能

1. **用户注册与登录**：新用户可以注册账号，已注册用户通过用户名和密码登录系统。
2. **图书浏览**：登录后可以查看图书馆所有藏书的信息，包括书名、作者、分类、出版社、出版年份、ISBN、可借数量等。
3. **图书搜索**：支持按书名、作者、ISBN、图书分类等关键字进行模糊搜索。
4. **图书借阅**：读者可以选择在库图书进行借阅操作，借阅期限默认为30天。系统会校验库存数量，确保可借数量大于0。同一读者不能重复借阅同一本未归还的图书。
5. **图书归还**：读者可以查看自己的借阅记录，并对借阅中的图书执行归还操作。
6. **借阅记录查看**：读者可以查看自己的全部借阅历史记录，包括借阅中、已归还和已逾期的图书。

#### 2.1.2 管理员功能

1. **图书管理**：管理员可以对图书进行增删改查操作，包括添加新书入库、编辑图书信息、删除已下架图书。
2. **借阅管理**：管理员可以查看所有读者的借阅记录，并可以代读者进行还书操作。
3. **用户管理**：系统区分读者和管理员角色，管理员登录后进入后台管理界面。

### 2.2 非功能需求

1. **易用性**：界面简洁直观，用户无需培训即可上手操作。
2. **可靠性**：系统能正确处理各类异常情况（如库存不足、重复借阅等），并给出友好的错误提示。
3. **安全性**：通过Session机制进行用户身份认证，未登录用户无法访问系统功能页面。
4. **可维护性**：采用MVC分层架构（Model-View-Controller），各层职责清晰，便于后期维护和扩展。
5. **响应式设计**：界面适配不同屏幕尺寸，支持移动端浏览。

## 四、详细设计说明

### 4.1 系统架构设计

系统采用经典的**MVC（Model-View-Controller）**三层架构模式：

- **Model层（模型层）**：包含实体类（`model`包）和数据访问类（`dao`包），负责数据的封装和数据库的CRUD操作。
- **View层（视图层）**：由JSP页面组成，负责展示数据和用户交互。使用JSTL标签库和EL表达式实现动态数据渲染。
- **Controller层（控制层）**：由Servlet组成，负责接收用户请求，调用业务逻辑，并将处理结果转发或重定向到视图层。

项目还包含一个**工具层（util包）**，封装了数据库连接和资源释放的通用操作。

### 4.3 核心业务流程设计

#### 4.3.1 用户认证流程

用户访问系统时，首先进入登录页面。输入用户名和密码后，系统在`users`表中查询匹配记录。若验证通过，将用户信息存入`HttpSession`；若验证失败，返回错误提示。所有功能页面在加载前均检查Session是否存在，未登录用户将被重定向到登录页。退出登录时销毁当前Session。

#### 4.3.2 图书借阅流程

1. 读者在图书列表页面点击"借阅"按钮。
2. 系统首先验证用户登录状态。
3. 系统检查目标图书的`available_copies`字段，确保库存大于0。
4. 系统检查`borrow_records`表中是否已存在该用户对该书的"借阅中"记录，防止重复借阅。
5. 验证通过后，创建借阅记录（`borrow_date`=当前日期，`due_date`=当前日期+30天，`status`='borrowed'），同时将图书的`available_copies`减1。

#### 4.3.3 图书归还流程

1. 读者在"我的借阅"页面点击"还书"按钮。
2. 系统验证借阅记录存在且状态为"borrowed"。
3. 更新借阅记录的`return_date`=当前日期，`status`='returned'。
4. 将对应图书的`available_copies`加1。

### 5.1 数据库连接工具类（DBUtil.java）

```java
package com.library.util;

import java.sql.*;

public class DBUtil {
    private static final String URL = "jdbc:mysql://localhost:3306/mobile?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf-8";
    private static final String USER = "root";
    private static final String PASSWORD = "root";

    static {
        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
        } catch (ClassNotFoundException e) {
            e.printStackTrace();
            throw new RuntimeException("MySQL驱动加载失败", e);
        }
    }

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(URL, USER, PASSWORD);
    }

    public static void close(Connection conn, Statement stmt, ResultSet rs) {
        try { if (rs != null) rs.close(); } catch (SQLException e) { e.printStackTrace(); }
        try { if (stmt != null) stmt.close(); } catch (SQLException e) { e.printStackTrace(); }
        try { if (conn != null) conn.close(); } catch (SQLException e) { e.printStackTrace(); }
    }

    public static void close(Connection conn, Statement stmt) {
        close(conn, stmt, null);
    }
}
```

**说明**：该工具类通过静态代码块加载MySQL驱动，提供获取数据库连接的静态方法，并封装了资源关闭方法，确保Connection、Statement、ResultSet按正确顺序释放，避免资源泄露。

### 5.5 登录Servlet（LoginServlet.java）

```java
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    protected void doPost(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        request.setCharacterEncoding("UTF-8");
        String username = request.getParameter("username");
        String password = request.getParameter("password");

        UserDao userDao = new UserDao();
        User user = userDao.login(username, password);

        if (user != null) {
            HttpSession session = request.getSession();
            session.setAttribute("user", user);
            response.sendRedirect("index");
        } else {
            request.setAttribute("error", "用户名或密码错误！");
            request.getRequestDispatcher("/jsp/login.jsp").forward(request, response);
        }
    }

    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        request.getRequestDispatcher("/jsp/login.jsp").forward(request, response);
    }
}
```

**说明**：使用`@WebServlet`注解配置Servlet映射，无需在web.xml中手动配置。POST请求处理登录逻辑，GET请求直接转发到登录页面。登录成功后将User对象存入Session，后续请求通过Session判断用户登录状态。采用重定向（sendRedirect）方式跳转到首页，避免表单重复提交。

### 5.6 借阅Servlet（BorrowServlet.java）

```java
@WebServlet("/borrow")
public class BorrowServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        HttpSession session = request.getSession(false);
        if (session == null || session.getAttribute("user") == null) {
            response.sendRedirect("login");
            return;
        }

        User user = (User) session.getAttribute("user");
        int bookId = Integer.parseInt(request.getParameter("bookId"));

        BookDao bookDao = new BookDao();
        Book book = bookDao.findById(bookId);

        // 校验1：图书存在且库存充足
        if (book == null || book.getAvailableCopies() <= 0) {
            request.getSession().setAttribute("msg", "该书库存不足，无法借阅！");
            response.sendRedirect("index");
            return;
        }

        BorrowDao borrowDao = new BorrowDao();
        // 校验2：未重复借阅
        if (borrowDao.hasBorrowed(user.getId(), bookId)) {
            request.getSession().setAttribute("msg", "您已经借阅过这本书且尚未归还！");
            response.sendRedirect("index");
            return;
        }

        // 借书：创建借阅记录 + 减少可用数量
        if (borrowDao.borrow(user.getId(), bookId)) {
            bookDao.decreaseAvailable(bookId);
            request.getSession().setAttribute("msg", "借阅成功！《" + book.getTitle() + "》，请在30天内归还。");
        } else {
            request.getSession().setAttribute("msg", "借阅失败，请重试！");
        }
        response.sendRedirect("index");
    }
}
```

**说明**：该Servlet完整实现了借阅业务的三重校验逻辑：①登录状态验证；②库存数量验证；③重复借阅验证。所有校验通过后才执行借阅操作。操作消息通过Session传递，在页面展示后立即清除（使用`<c:remove>`标签），实现一次性提示效果。

```java
        request.setCharacterEncoding("UTF-8");
        String action = request.getParameter("action");
        BookDao bookDao = new BookDao();

        if ("add".equals(action)) {
            // 封装Book对象并调用add方法
            Book book = new Book();
            book.setTitle(request.getParameter("title"));
            book.setAuthor(request.getParameter("author"));
            // ... 设置其他字段
            if (bookDao.add(book)) {
                session.setAttribute("msg", "图书添加成功！");
            }
        } else if ("update".equals(action)) {
            // 封装Book对象并调用update方法
            // ...
        } else if ("delete".equals(action)) {
            int id = Integer.parseInt(request.getParameter("id"));
            if (bookDao.delete(id)) {
                session.setAttribute("msg", "图书删除成功！");
            }
        }
        response.sendRedirect("books");
    }
}
```

**说明**：管理员Servlet中每个方法都先进行管理员权限验证，通过一个Servlet处理图书的增删改三种操作，通过`action`参数区分具体操作类型。这是一种常见的设计模式，减少了Servlet数量，使代码结构更加紧凑。

### 7.3 调试过程中遇到的错误及解决方案

**问题1：MySQL驱动类找不到（ClassNotFoundException: com.mysql.cj.jdbc.Driver）**

- **错误现象**：启动应用后，首次访问数据库时报错，提示找不到MySQL驱动类。
- **原因分析**：pom.xml中已添加`mysql-connector-java`依赖，但在DBUtil中使用的驱动类名为`com.mysql.cj.jdbc.Driver`（MySQL 8.x版本），而`Class.forName()`加载时如果依赖未被正确引入到classpath中就会报此错误。
- **解决方案**：确认pom.xml中的依赖scope不是`provided`（Jetty运行时需要此依赖），确保`mvn jetty:run`正确解析了所有依赖。检查MySQL版本，对于MySQL 8.0使用`com.mysql.cj.jdbc.Driver`；若使用MySQL 5.x，则需改为`com.mysql.jdbc.Driver`。

**问题2：JSP页面EL表达式不解析，直接显示${...}原文**

- **错误现象**：页面中${book.title}等表达式没有被解析，而是直接显示为文本。
- **原因分析**：JSP页面的`page`指令中可能缺少`isELIgnored="false"`属性，或者web.xml中Servlet版本配置过低。JSP 2.0以上默认启用EL，但某些情况下可能被禁用。
- **解决方案**：检查web.xml，确认使用Servlet 4.0版本（`version="4.0"`），确保`web-app`命名空间正确。在JSP页面顶部添加`<%@ page isELIgnored="false" %>`显式声明启用EL。

**问题3：数据库中文乱码**

- **错误现象**：从数据库读取的中文数据显示为????或乱码。
- **原因分析**：数据库连接URL中缺少字符编码参数，或者数据库/表的默认字符集不是UTF-8。
- **解决方案**：在JDBC连接URL中添加`characterEncoding=utf-8`参数，同时在`init.sql`中创建数据库时指定`DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci`，确保从数据库层到连接层全链路使用UTF-8编码。

**问题4：表单提交后刷新页面导致重复提交**

- **错误现象**：添加图书后刷新页面，图书被重复添加。
- **原因分析**：表单POST提交后使用了`forward`方式转发，浏览器地址栏仍显示表单提交URL，用户刷新时会重新发送POST请求。
- **解决方案**：在Servlet的POST处理完成后，使用`response.sendRedirect()`进行重定向跳转，将浏览器地址栏变为GET请求的URL，从而遵循Post-Redirect-Get（PRG）设计模式，避免表单重复提交。

**问题5：管理员后台页面未做权限检查导致普通用户可访问**

- **错误现象**：普通读者直接在浏览器输入`/admin/books`即可访问管理页面。
- **原因分析**：AdminBookServlet和AdminBorrowsServlet最初未检查用户角色，只要登录就可访问。
- **解决方案**：在每个管理员Servlet的`doGet`和`doPost`方法开头增加管理员权限验证代码，检查`user.isAdmin()`，非管理员用户直接重定向到读者首页。

**问题6：CSS样式文件路径在子目录Servlet下失效**

- **错误现象**：管理员页面（URL为`/admin/books`）加载不到CSS文件。
- **原因分析**：CSS引用路径使用了相对路径`css/style.css`，当页面URL为`/library/admin/books`时，浏览器会将CSS请求解析为`/library/admin/css/style.css`，而不是正确的`/library/css/style.css`。
- **解决方案**：在所有JSP页面中使用`${pageContext.request.contextPath}/css/style.css`作为CSS路径，使用绝对路径确保在任何页面层级下都能正确加载。

## 八、课程实训总结

### 8.1 项目中用到的知识点

通过本次JavaWeb期末大作业的开发实践，系统运用了本课程所学的多项核心技术：

1. **Servlet技术**：使用HttpServlet处理HTTP请求和响应，通过`@WebServlet`注解配置URL映射，理解并运用了Servlet的生命周期（init、service、destroy）、`doGet()`和`doPost()`方法的请求分发机制。

2. **JSP技术**：使用JSP作为视图层模板引擎，掌握JSP九大内置对象（特别是request、response、session、application），使用JSP脚本元素和EL表达式进行数据展示。

3. **JSTL标签库**：使用核心标签库（`<c:forEach>`、`<c:if>`、`<c:choose>`、`<c:remove>`等）实现页面的逻辑控制和数据迭代，减少Java脚本在JSP中的使用，使视图层代码更加简洁清晰。

4. **JDBC数据库编程**：使用JDBC API连接MySQL数据库，掌握Connection、PreparedStatement、ResultSet的使用方式。使用PreparedStatement预编译SQL语句有效防止SQL注入，正确使用try-catch-finally块确保数据库资源及时释放。

5. **MVC设计模式**：将系统分为Model（实体类和DAO）、View（JSP页面）、Controller（Servlet）三层，实现了业务逻辑、数据访问和页面展示的分离，提高了代码的可维护性和可扩展性。

6. **Session会话管理**：使用HttpSession实现用户登录状态的保持，通过`session.setAttribute()`存储用户信息，通过`session.invalidate()`实现安全退出。

7. **请求转发与重定向**：理解`request.getRequestDispatcher().forward()`（服务器内部转发，URL不变，可携带request属性）和`response.sendRedirect()`（客户端重定向，URL改变，通过session传递数据）的区别和使用场景。

8. **Maven项目管理**：使用Maven进行项目依赖管理和构建，通过pom.xml配置项目依赖（Servlet API、MySQL Connector、Jetty插件），理解Maven的标准目录结构。

9. **嵌入式Jetty服务器**：使用jetty-maven-plugin作为开发测试用Web服务器，无需安装配置Tomcat，一条命令即可启动应用，大大简化了开发环境的搭建。

### 8.2 程序有待改进的地方

1. **密码明文存储**：当前系统中用户密码以明文形式存储在数据库中，存在安全隐患。后续应引入MD5或BCrypt等加密算法对密码进行哈希处理。

2. **缺少事务管理**：借书操作涉及两个步骤（创建借阅记录和减少库存数量），归还操作类似。当前代码中这两步操作没有放在数据库事务中，如果第二步失败，第一步不会回滚，可能导致数据不一致。后续可以使用JDBC的事务管理（`conn.setAutoCommit(false)`和`conn.commit()/rollback()`）来保证操作的原子性。

3. **缺少输入验证**：当前对用户输入的数据验证较为简单（仅前端HTML5的required属性），应在服务端增加更严格的数据校验，如用户名长度限制、密码强度验证、手机号格式校验等。

4. **分页功能缺失**：图书列表和借阅记录列表没有分页功能，当数据量增大时，页面加载速度和用户体验都会受到影响。后续可以引入分页查询（使用LIMIT和OFFSET）来优化。

5. **逾期处理机制不完善**：虽然数据库设计了`overdue`状态，但当前系统缺少自动检测和标记逾期记录的功能。可以通过定时任务（如Java Timer或Spring的@Scheduled）每日扫描超过应还日期且未归还的记录，自动更新状态为"overdue"。

6. **前端可进一步优化**：当前前端使用原生CSS和JavaScript，后续可以引入Bootstrap等前端框架提升界面美观度和交互体验，或使用AJAX技术实现页面无刷新操作。

7. **未使用连接池**：当前每次数据库操作都创建新连接，高并发场景下性能较差。后续可以引入C3P0或Druid等数据库连接池。
